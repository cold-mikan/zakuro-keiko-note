# Notion同期の設定手順

このアプリでは、稽古出欠ノートの稽古日・出欠情報をNotionの「稽古カレンダー」データベースへ一方通行で同期できます。

編集の本体は稽古出欠ノートです。Notionは確認用カレンダーとして使います。

## 1. Notionにデータベースを作る

Notionで「稽古カレンダー」というデータベースを作り、以下のプロパティを追加してください。

| プロパティ名 | 種類 |
| --- | --- |
| 名前 | タイトル |
| 日付 | 日付 |
| 開始 | テキスト |
| 終了 | テキスト |
| 場所 | テキスト |
| 予定の種類 | セレクト |
| 対象チーム | セレクト |
| 出席数 | 数値 |
| 欠席数 | 数値 |
| 遅刻数 | 数値 |
| 早退数 | 数値 |
| 未定数 | 数値 |
| 未回答数 | 数値 |
| 出席者 | テキスト |
| 欠席者 | テキスト |
| 遅刻者 | テキスト |
| 早退者 | テキスト |
| 未回答者 | テキスト |
| この日にやるシーン | テキスト |
| メモ | テキスト |
| 稽古ノートID | テキスト |

`稽古ノートID` は重複防止に使います。削除しないでください。

## 2. Notion Integrationを作る

1. Notionの開発者ページでIntegrationを作成します。
2. Internal Integration Tokenを控えます。
3. Notionの「稽古カレンダー」データベースを開きます。
4. 右上のメニューから、作成したIntegrationを接続します。

## 3. データベースIDを確認する

NotionのデータベースURLからIDを控えます。

例：

```text
https://www.notion.so/xxxx?v=yyyy
```

URL内の長い英数字部分がデータベースIDです。

## 4. Vercelに環境変数を追加する

Vercelの `Environment Variables` に以下を追加してください。

```text
NOTION_TOKEN=NotionのInternal Integration Token
NOTION_REHEARSAL_DATABASE_ID=稽古カレンダーのデータベースID
```

追加後、VercelでRedeployしてください。

## 5. 同期する

アプリの `出欠一覧` 画面にある `Notionへ同期` ボタンを押すと、全稽古日がNotionへ送られます。

同じ稽古日は `稽古ノートID` を使って既存ページを更新するので、何度押しても重複しにくい作りです。
